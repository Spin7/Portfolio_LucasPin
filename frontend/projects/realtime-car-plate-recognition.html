Realtime Car Plate Recognition

Year: 2025

Overview

A complete end-to-end pipeline for real-time vehicle license plate detection and OCR using YOLOv8 (exported to ONNX) and EasyOCR, designed to run on CPU or CUDA without requiring PyTorch at inference time. Each plate crop is preprocessed into 4 image variants and read with weighted voting and a position-aware correction step that enforces the LLLLNNN plate format. A temporal tracker with ghost tracks and majority vote confirmation keeps text stable on screen for ~1.5 s after a plate disappears. The composite UI shows an annotated video pane alongside a scrollable plate panel with live FPS stats.

Inference pipeline — video mode

Detect — each frame is fed to the ONNX detector, which returns bounding boxes around any visible plate  ·  Crop & rectify — each plate region is extracted with a quality margin and perspective-corrected  ·  OCR — four preprocessed variants are read in parallel and the best text is selected by weighted vote  ·  Track — plates are matched across frames by centroid proximity; confirmed text is locked in by majority vote and kept visible during brief disappearances  ·  Render — annotated video and a live plate panel are composited side-by-side and written to disk

YOLO for Plate Localisation

YOLOv8 — Training & Export

The detector is built on YOLOv8, trained from scratch on a 10 125-image dataset sourced from Roboflow. An Optuna hyperparameter search over 15 parameters (learning rate, momentum, augmentation factors, loss weights…) ran for 10 trials maximising mAP50, after which a final 270-epoch training run was executed with the best configuration found.

ONNX Export for Runtime Efficiency

After training, the model is exported to ONNX format (opset 12, input size 224×224 px). This decouples inference from PyTorch entirely — the detector runs via ONNX Runtime on either CPU or CUDA, making the system lightweight and easy to deploy without a full deep-learning stack.

  • Letterbox preprocessing — frames are resized preserving aspect ratio and padded to a square before inference.
  • Pure-NumPy NMS — Non-Maximum Suppression is applied in NumPy after decoding the raw output tensor, keeping the runtime dependency-free.
  • Auto device selection — CUDA is detected at startup; the system falls back to CPU transparently.

OCR Detection

Multi-Variant OCR with Weighted Voting

Once a plate is localised, the crop is fed through a robust multi-variant OCR pipeline powered by EasyOCR. Because plate images in real traffic suffer from motion blur, low contrast, and variable lighting, each crop is preprocessed into four distinct grayscale variants before being read:

  • CLAHE + denoising — enhances local contrast while suppressing noise.
  • Adaptive threshold — binarises the image locally, handling uneven illumination.
  • Grayscale–binary fusion — blends both representations to balance detail and crispness.
  • Otsu + dilation — global threshold followed by morphological dilation to strengthen thin strokes.

Format Correction & Perspective Rectification

All variants are upscaled 2× before OCR to improve character resolution. Results are combined with a weighted vote + 7-character length bonus to pick the best reading. A final position-aware correction step maps common OCR confusions to the expected LLLLNNN format (e.g. 0→O in letter slots, O→0 in digit slots). Tilted plates are straightened via perspective rectification before the variants are generated.
