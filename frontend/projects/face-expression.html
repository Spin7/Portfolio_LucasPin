VideoCall Behavior Analysis

Year: 2025

Overview

VideoCall Behavior Analysis is a fully local system that analyses the behaviour of up to nine video-call participants in real time — with no cloud dependency. A custom YOLOv8 face detector, exported to ONNX and running on CPU, locates every face in the frame; a MediaPipe Face Landmarker (478 points) then drives eye, mouth, smile/grimace and yawn detection together with iris-based gaze tracking, while a MediaPipe Hand Landmarker recognises gestures such as thumbs-up and victory signs. Each participant is re-identified across frames using a custom multi-region LBPH + HSV descriptor, matched through cosine similarity with an SVD subspace residual fallback. Every metric is smoothed with EMA filters and rendered as a live 9-tile mosaic with a per-person status panel.

Per-frame pipeline — detect, identify, analyse, render

Detect — caritas.onnx (YOLOv8) locates every face in the frame  ·  Identify — multi-region LBPH + HSV embedding matched against the Face Gallery (cosine + SVD residual)  ·  Analyse — MediaPipe Face/Hand Landmarker drives eyes, mouth, smile/grimace, yawn, gaze and gestures  ·  Smooth — every metric passes through an EMA filter to reduce flicker  ·  Render — per-person tiles are assembled into the live 9-tile mosaic

Face Gallery & Identity Enrollment

Three Enrollment Modes

The Face Gallery stores the reference embeddings used to recognise each participant across frames. Three modes cover different scenarios, selectable from a CLI menu at startup.

  • Mode A — Folder (best accuracy) — one subfolder per person under Face Gallery/; the system detects a face in each image and builds an SVD subspace per person.
  • Mode B — Live multi-frame (best for real-time sources) — a live preview window records 30 frames per person while they naturally roll and turn their head, capturing pose variation a single photo can't.
  • Mode C — Single-frame (legacy / fastest) — grabs one frame, detects every face, and assigns names through console prompts.
  • Mode S — Skip — reuses a gallery already persisted from a previous session.

Whichever mode is used, the gallery is saved automatically to face_gallery2.pkl and reloaded at the next startup — enrollment only needs to happen once per person.

Multi-Region Descriptor & Matching

Each enrolled face is resized to a fixed 96×96 crop and divided into a 6×6 grid of regions. Every cell contributes a 32-bin LBP texture histogram and an HSV colour histogram, blended 80% texture / 20% colour into a single descriptor — combining fine skin-texture detail with coarse colour cues that survive small lighting changes.

At match time, the descriptor is compared against each person's stored embeddings (up to 50 per person) using cosine similarity, averaging the top-3 closest matches. If the score sits near the 0.72 acceptance threshold, an SVD subspace residual check (threshold 0.20) acts as a second opinion before confirming or rejecting the identity.
