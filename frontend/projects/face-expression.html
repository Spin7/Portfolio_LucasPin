VideoCall Behavior Analysis

Year: 2025

Overview

VideoCall Behavior Analysis is a fully local system that analyses the behaviour of up to nine video-call participants in real time — with no cloud dependency. A custom YOLOv8 face detector, exported to ONNX and running on CPU, locates every face in the frame; a MediaPipe Face Landmarker then drives eye, mouth, smile/grimace and yawn detection together with iris-based gaze tracking, while a MediaPipe Hand Landmarker recognises gestures such as thumbs-up and victory signs. Each participant is re-identified across frames using a custom multi-region LBPH + HSV descriptor, matched through cosine similarity with an SVD subspace residual fallback. Every metric is smoothed with EMA filters and rendered as a live 9-tile mosaic with a per-person status panel.

Detect — a custom YOLOv8 model locates every face in the frame  ·  Identify — multi-region LBPH + HSV embedding matched against enrolled participants (cosine + SVD residual)  ·  Analyse — MediaPipe Face/Hand Landmarker drives eyes, mouth, smile/grimace, yawn, gaze and gestures  ·  Smooth — every metric passes through an EMA filter to reduce flicker  ·  Render — per-person tiles are assembled into the live 9-tile mosaic

Face Gallery & Identity Enrollment

Three Enrollment Modes

The Face Gallery stores the reference embeddings used to recognise each participant across frames. Three modes cover different scenarios, selectable from a CLI menu at startup.

  • Mode A — Folder (best accuracy) — organise a folder of photos per person; the system detects a face in each image and builds an SVD subspace per person.
  • Mode B — Live multi-frame (best for real-time sources) — a live preview window records a short burst of frames per person while they naturally roll and turn their head, capturing pose variation a single photo can't.
  • Mode C — Single-frame (legacy / fastest) — grabs one frame, detects every face, and assigns names through console prompts.
  • Mode S — Skip — reuses a gallery already persisted from a previous session.

Whichever mode is used, the gallery is saved automatically and reloaded at the next startup — enrollment only needs to happen once per person.

Multi-Region Descriptor & Matching

Each enrolled face is resized to a fixed crop and divided into a small grid of regions. Every region contributes a local LBP texture histogram and an HSV colour histogram, blended together into a single descriptor — combining fine skin-texture detail with coarse colour cues that survive small lighting changes.

At match time, the descriptor is compared against each person's stored embeddings using cosine similarity, averaging the closest matches for robustness. When the score sits near the acceptance boundary, an SVD subspace residual check acts as a second opinion before confirming or rejecting the identity.

MediaPipe Landmarkers

Face Landmarker — Facial Mesh

Every face crop returned by the detector is passed to MediaPipe's Face Landmarker, which fits a dense facial mesh — the standard face mesh plus a set of refined iris landmarks. The model ships as a lightweight, self-contained package, so no extra download is needed at runtime.

The system reads specific subsets of this mesh — the eye contours for an EAR (eye-aspect ratio), the lip contour for a MAR (mouth-aspect ratio) and lip curvature, and the iris centres for gaze displacement — rather than the mesh as a whole. The model is configured to focus on a single, dominant face inside each cropped tile, since only one participant appears per tile in the mosaic layout.

Hand Landmarker — Hand Mesh

Gesture recognition uses MediaPipe's Hand Landmarker, a lighter model that tracks the wrist, palm, and finger joints — for up to two hands simultaneously. Like the face model, it ships as a compact, self-contained package that needs no extra download.

The system compares the relative position of each fingertip to its base joint to tell an extended finger from a curled one, then combines that per-finger state into a gesture: Thumbs Up (thumb extended, others curled), Victory (index and middle extended in a V), or a generic Detected status when a hand is present but no specific gesture matches.

Per-Frame Pipeline

Every face that ends up on the mosaic went through the same chain of steps first — run on every single frame: detect it, try to recognise it, and — whether that recognition succeeds or not — read its behavioural signals anyway. Here's that journey, step by step.

⏲ Per frame
1

Input frame

A new frame is read from whichever source is active — camera, video file, or screen capture.

2

Face detection (ONNX)

The frame is resized to a fixed input size and passed through a lightweight, CPU-only detector. Low-confidence hits are dropped and overlapping boxes are merged, leaving a clean list of candidate faces.

3

Crop & descriptor

Each face is cropped using exactly its detected box — not yet expanded. That crop is normalised for size and lighting and turned into a compact descriptor that blends local texture with colour.

4

Identity matching

The descriptor is compared against everyone enrolled in the gallery in two passes: a fast similarity check, then a stricter fallback check for borderline cases. If neither confirms an identity, the face carries on as unidentified — it doesn't leave the pipeline.

↳ No match found — continues as "Unknown", same path as everyone else
👤 Per detected face
5

Expand & smooth the crop

The tight face box is grown outward to approximate the full call tile around the person, using edge and colour cues against the background. That larger crop is then smoothed frame-to-frame so it doesn't jitter.

6

Face & hand landmarks

The expanded crop — not the raw detection crop — is fed to the face and hand landmark models. This runs for every detected face, identified or not, since reading behaviour doesn't depend on knowing who someone is.

7

Metric extraction

From those landmarks the system derives, all at once: eye state, mouth state, smile/grimace, sustained yawning, gaze/focus, and hand gesture.

8

Final mosaic gate

Only faces successfully matched to a real identity earn a visible tile on the mosaic. Unidentified faces were fully analysed anyway — they simply aren't shown. Every frame does more work than the final mosaic reveals.

✓ Shown on mosaic — Analysed, not shown
