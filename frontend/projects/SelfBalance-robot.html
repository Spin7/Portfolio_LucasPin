Self-Balancing Robot via State Feedback Control

Year: 2024

Overview

Self-Balancing Robot via State Feedback Control is an academic project featuring a full plant model of a two-wheeled inverted pendulum on wheels, built through physical modelling → Lagrange-Euler derivation → discrete LQR design → MATLAB simulation → Arduino embedded implementation. A full state-feedback controller running at 100 Hz on an Arduino Uno, fed by an MPU-6050 IMU and optical tachometers through an L298N driver, keeps the robot balanced in real time.

Physical Model & Lagrange-Euler Derivation

Fig. 1 — Simplified 2D scheme. Generalised coordinates: x (horizontal axle position) and θ (pendulum angle from vertical).

Plant Description

The robot is modelled as a system of coupled rigid bodies: two hollow cylinders (wheels) that roll without slipping, plus three rigid prismatic layers stacked above the wheel axle (reference \(y=0\)):

  • Prism 1 — Battery + Motors + Tachometers  |  \(m_1 = 0.283\;\text{kg}\), \(h_1 = 0.001\;\text{m}\)
  • Prism 2 — Arduino Uno + L298N driver  |  \(m_2 = 0.050\;\text{kg}\), \(h_2 = 0.052\;\text{m}\)
  • Prism 3 — MPU-6050 IMU  |  \(m_3 = 0.002\;\text{kg}\), \(h_3 = 0.100\;\text{m}\)

The two generalised coordinates are \(q_1 = x\) (horizontal position of the wheel axle) and \(q_2 = \theta\) (pendulum angle measured from the vertical), reducing the 3-D dynamics to a tractable 2-D problem.

Kinetic Energy — Wheels

Each wheel (mass \(m_r/2\)) rolls without slipping (\(\dot x = r\dot\varphi\)). Translational plus rotational contributions (\(I_r = \tfrac{1}{2}\cdot\tfrac{m_r}{2}(r^2+r_\text{int}^2)\)):

\[ T_\text{wheels} = 2\!\left[ \frac{1}{2}\cdot\frac{m_r}{2}\,\dot x^2 + \frac{1}{2}\cdot I_r\!\left(\frac{\dot x}{r}\right)^{\!2} \right] = \frac{3}{4}\,m_r\,\dot x^2 \]

Kinetic Energy — Prismatic Layers

For prism \(i\): position \(x_i = x + h_i\sin\theta\), \(y_i = h_i\cos\theta\), so the squared centre-of-mass speed is:

\[ v_i^2 = \dot x^2 + 2h_i\,\dot x\,\dot\theta\cos\theta + h_i^2\,\dot\theta^2 \]

With \(m_T = \sum m_i\), \(M_h = \sum m_i h_i\), \(I_h = \sum m_i h_i^2\):

\[ T_\text{prisms} = \frac{1}{2}\!\left( m_T\,\dot x^2 + 2M_h\,\dot x\,\dot\theta\cos\theta + I_h\,\dot\theta^2 \right) \]

Total Kinetic Energy & Lagrangian

Defining \(M = m_T + \tfrac{3}{2}m_r\):

\[ T = \frac{1}{2}M\,\dot x^2 + M_h\,\dot x\,\dot\theta\cos\theta + \frac{1}{2}I_h\,\dot\theta^2 \]

Potential energy (gravitational, \(C_1 = g\,M_h\)):

\[ V = \sum_{i=1}^{3} m_i\,h_i\,g\cos\theta = C_1\cos\theta \]

The Lagrangian \(\mathcal{L} = T - V\):

\[ \mathcal{L} = \frac{1}{2}M\,\dot x^2 + M_h\,\dot x\,\dot\theta\cos\theta + \frac{1}{2}I_h\,\dot\theta^2 - C_1\cos\theta \]

Euler-Lagrange Equations

Applying \(\dfrac{d}{dt}\!\left(\dfrac{\partial\mathcal{L}}{\partial\dot q_k}\right) -\dfrac{\partial\mathcal{L}}{\partial q_k} = Q_k\) for each coordinate (linear friction \(b\), angular friction \(c\)):

\[ \begin{aligned} (1)\quad & M\ddot x + M_h\cos\theta\,\ddot\theta = F - b\dot x + M_h\sin\theta\,\dot\theta^2 \\[8pt] (2)\quad & I_h\ddot\theta + M_h\cos\theta\,\ddot x = C_1\sin\theta - c\dot\theta \end{aligned} \]

Linearisation around \(\theta = 0\)

Small-angle approximations \(\cos\theta \approx 1\), \(\sin\theta \approx \theta\), \(\dot\theta^2 \approx 0\):

\[ \begin{aligned} (1')\quad & M\ddot x + M_h\ddot\theta = F - b\dot x \\[8pt] (2')\quad & I_h\ddot\theta + M_h\ddot x = C_1\theta - c\dot\theta \end{aligned} \]

State-Space Representation

State vector \(\mathbf{X} = \begin{bmatrix} x & \dot x & \theta & \dot\theta \end{bmatrix}^T\). Solving \((1')\)–\((2')\) for \(\ddot x\) and \(\ddot\theta\) (with \(\Delta = MI_h - M_h^2\), \(u = F\)):

\[ \begin{aligned} \ddot x &= -\frac{I_h b}{\Delta}\,\dot x +\frac{M_h C_1}{\Delta}\,\theta -\frac{M_h c}{\Delta}\,\dot\theta +\frac{I_h}{\Delta}\,u \\[8pt] \ddot\theta &= +\frac{M_h b}{\Delta}\,\dot x +\frac{M C_1}{\Delta}\,\theta -\frac{M c}{\Delta}\,\dot\theta -\frac{M_h}{\Delta}\,u \end{aligned} \]

The continuous-time model \(\dot{\mathbf{X}} = A\mathbf{X} + Bu\):

\[ A = \begin{bmatrix} 0 & 1 & 0 & 0 \\[6pt] 0 & -\dfrac{I_h b}{\Delta} & \dfrac{M_h C_1}{\Delta} & -\dfrac{M_h c}{\Delta} \\[12pt] 0 & 0 & 0 & 1 \\[6pt] 0 & \dfrac{M_h b}{\Delta} & \dfrac{M C_1}{\Delta} & -\dfrac{M c}{\Delta} \end{bmatrix} \qquad B = \begin{bmatrix} 0 \\[6pt] \dfrac{I_h}{\Delta} \\[12pt] 0 \\[6pt] -\dfrac{M_h}{\Delta} \end{bmatrix} \]

Substituting the physical parameters and discretising by ZOH at \(T_s = 10\text{ ms}\) yields the discrete matrices \(G\) and \(H\) used in the LQR design.

Controller Design — Discrete LQR

Design Choice: Direct State Feedback

Although an observer-based servo was considered, it was discarded because linear position x cannot be measured directly — it requires numerical integration of tachometer readings, introducing accumulated error. A simpler direct state-feedback law was chosen.

LQR Gain Computation

The feedback matrix K minimises the infinite-horizon quadratic cost:

J = sum( X[k]' Q X[k] + u[k]' R u[k] )

Q = diag([200, 10, 200, 20])
R = 1
K = dlqr(G, H, Q, R)

K = [-4.7152,  -3.3278,  -17.0580,  -1.8229]

Closed-Loop Difference Equation

X[k+1] = G*X[k] + H*u[k]
u[k]   = -K*X[k]

Motor Direction Logic

u >  1  -> both motors BACKWARD
u < -1  -> both motors FORWARD
|u| ≤ 1 -> both motors STOP

PWM magnitude is clamped to [0, 230] (out of 255).

ẋ — linear velocity Avg. wheel angular velocity × r = 0.034 m

MATLAB Simulation — Discomp_SelfBalanceRobot.m

The script Discomp_SelfBalanceRobot.m implements the complete design pipeline in four sequential sections: discretisation of the continuous plant, structural verification (controllability and observability), LQR gain computation, and closed-loop time-domain simulation from a 10° initial tilt. All four state trajectories are plotted to verify controller performance.

§ 2 — Discretisation (ZOH, Ts = 10 ms)

The continuous-time plant \((A,B,C,D)\) is converted to a discrete-time model \((G,H)\) using Zero-Order Hold (ZOH), which faithfully represents the constant-between-samples nature of a microcontroller's PWM output: 

Ts = 0.01;                       % Sampling period: 10 ms
sys_C = ss(A, B, C, D);          % Continuous plant object
sys_D = c2d(sys_C, Ts, 'zoh');   % ZOH discretisation

G = sys_D.A;   % Discrete state matrix
H = sys_D.B;   % Discrete input matrix

§ 3 — Controllability & Observability Check

Before proceeding to controller design, the script verifies that the discrete system is completely controllable (all states can be driven to any target) and completely observable (all states can be inferred from the output \(y = x\)). Both conditions are confirmed when the rank of the respective structural matrices equals the system order \(n = 4\): 

if rank(ctrb(G, H)) < n
    error('The Discrete System is NOT controllable');
end
if rank(obsv(G, C)) < n
    error('The Discrete System is NOT observable');
end

Both checks pass for the physical parameters of this robot, confirming that a state-feedback law \(u = -K\mathbf{X}\) is well-defined and that the single output \(x\) is sufficient to reconstruct the full state (should an observer be needed).

§ 4 — State Feedback Design (Discrete LQR)

The optimal gain matrix \(K\) is obtained by minimising the infinite-horizon quadratic cost \(J = \sum_k (\mathbf{X}^T Q\,\mathbf{X} + u^T R\,u)\). High weights on \(x\) and \(\theta\) penalise position and tilt errors strongly: 

Q = diag([200, 10, 200, 20]);  % High penalty on x (x1) and theta (x3)
R = 1;                         % Penalty on control effort u

K = dlqr(G, H, Q, R);
% K = [-4.7152,  -3.3278,  -17.0580,  -1.8229]

The weight matrix \(Q = \text{diag}(200, 10, 200, 20)\) assigns the highest penalty to linear position \(x_1\) and pitch angle \(x_3\) — the two states most critical for balance — while keeping moderate penalties on their rates. \(R = 1\) is a neutral penalty on control effort, allowing the actuator to act aggressively when needed.

§ 5 — Closed-Loop Simulation

The closed-loop difference equation \(\mathbf{X}[k+1] = G\,\mathbf{X}[k] + H\,u[k]\), \(u[k] = -K\,\mathbf{X}[k]\) is iterated for \(T_\text{total} = 3\) s starting from an initial tilt of \(\theta_0 = 10^\circ\) with all other states at zero: 

T_total = 3;                         % Simulation duration [s]
N       = T_total / Ts;              % Number of steps

initial_angle = (pi/180) * 10;       % 10 deg initial tilt
X = [0;  0;  initial_angle;  0];     % Initial state vector

for k = 1:N
    u          = -K * X(:,k);        % LQR control law
    X(:,k+1)  = G * X(:,k) + H * u; % Discrete state update
end

All four states converge to zero within approximately 2–3 seconds, confirming that the LQR design successfully stabilises the inverted-pendulum dynamics from a 10° initial tilt. The absence of steady-state error in angle and velocity demonstrates the effectiveness of the chosen \(Q\) and \(R\) weights.

Embedded Implementation & Custom C++ Classes

Hardware Architecture

  • Arduino Uno — main microcontroller running the 10 ms control loop
  • MPU-6050 — 6-DOF IMU via I²C @ 400 kHz; pitch angle and rate
  • 2× Optical Tachometers — interrupt-driven, 20 pulses/rev
  • L298N Dual H-Bridge — accepts PWM, drives both DC motors
  • 2× DC Motors — wheel actuation
  • LiPo Battery (125 g) — compact power supply

Control Loop

The control loop runs at a nominal 10 ms period enforced by a millis() timer gate. The control action u is computed and applied only when 10 ms have elapsed. 

// Control law (Arduino C++)
u = -( x*k1 + v_lineal*k2 + Pitch_angle*k3 + Pitch_rate*k4 );
// k = [-4.7152, -3.3278, -17.0580, -1.8229]

Results & Demo

The robot “Paco” successfully self-balances in place and recovers from small manual disturbances. Pitch angle data logged via Serial confirms the system oscillates around θ ≈ 0° within the sensor noise floor. The absence of integral action results in a small but bounded steady-state position drift — a known characteristic of pure state-feedback. The LQR controller drives all four states to equilibrium within 2–3 seconds from a 10° initial tilt in MATLAB simulation.
